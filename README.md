# vacancy-parser

Telegram-бот, который автоматически собирает вакансии для исследователей (UX, CX, Product, Market Research) из разных источников и публикует их в Telegram-каналы.

**Два канала:**
- 🇷🇺 **РФ** — русскоязычные вакансии с hh.ru (и позже из Telegram-каналов)
- 🌍 **Global** — англоязычные вакансии с Remotive, Himalayas, Arbeitnow (ещё не подключены)

**ЦА:** UX, CX, Product, Market ресёрчеры. Грейд: от Intern до Lead.

---

## Что сейчас работает

**Парсер hh.ru** — двухшаговый:
1. Ищет вакансии по 29 поисковым запросам (на русском и английском)
2. По каждой найденной вакансии запрашивает полную карточку (описание, навыки)

**Фильтрация** — подход «белый список»:
- Вакансия проходит, только если в заголовке есть одно из ключевых слов: `исследователь`, `researcher`, `research`, `ux`, `cx`, `insight`, `usability`, `юзабилити`, `исследования`
- Короткий чёрный список отсекает ложные срабатывания (инженер-исследователь, UX/UI-дизайнер и т.п.)
- Это проще и надёжнее, чем 150 стоп-фраз, которые мы пробовали раньше

**Результат последнего прогона:** ~78 релевантных вакансий за 7 дней.

**Что ещё не подключено:** Telegram-бот (код написан, нужны токен и каналы), планировщик, глобальные источники.

## Быстрый старт (macOS)

### 1. Установить инструменты

Открой **Terminal** (Cmd+Пробел → набери "Terminal" → Enter).

**Homebrew** (менеджер пакетов, как App Store для программистов):
```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```
После установки Homebrew попросит выполнить 2-3 команды из раздела "Next steps" — выполни их.

**Python и Poetry:**
```bash
brew install python@3.12
brew install poetry
```

Проверка:
```bash
python3 --version    # должно показать 3.12.x
poetry --version     # должно показать 2.x
```

### 2. Скачать проект

```bash
cd ~/Desktop
git clone <URL-ВАШЕГО-РЕПО>
cd vacancy-parser
```

### 3. Установить зависимости

```bash
poetry install
```

Эта команда создаёт изолированную среду Python и ставит все нужные библиотеки. Ничего на маке не сломает.

### 4. Настроить секреты

```bash
cp .env.example .env
```

Открой `.env` в любом текстовом редакторе и заполни:

```
TELEGRAM_BOT_TOKEN=123456:ABC-DEF...    ← от @BotFather в Telegram
TELEGRAM_CHANNEL_RU=-100xxxxxxxxxx      ← ID РФ-канала
TELEGRAM_CHANNEL_GLOBAL=-100xxxxxxxxxx  ← ID глобального канала
```

**Как получить токен бота:**
1. В Telegram найди @BotFather → напиши `/newbot`
2. Дай имя и username → получишь токен

**Как получить ID канала:**
1. Создай канал → добавь бота как админа (с правом писать)
2. Перешли любое сообщение из канала боту @userinfobot → он покажет ID

### 5. Создать базу данных

```bash
poetry run python main.py --init-db
```

### 6. Запустить

```bash
poetry run python main.py --once    # один прогон (для теста)
poetry run python main.py           # полный режим (каждые 30 мин)
```

Остановить: `Ctrl+C`.

## Как работает парсинг

```
  hh.ru API
      │
      ▼
  29 поисковых запросов ──→ Список вакансий (ID + сниппет)
      │
      ▼
  По каждой: запрос полной карточки ──→ Описание + навыки
      │
      ▼
  Белый список: есть "researcher" / "исследователь" / "ux" / "cx" в заголовке?
      │                                           │
     Да                                          Нет → отсеиваем
      │
      ▼
  Чёрный список: это "инженер-исследователь" или "UX/UI-дизайнер"?
      │                                           │
     Нет                                         Да → отсеиваем
      │
      ▼
  ✅ Вакансия прошла → сохраняется в базу → постится в канал
```

## Как настраивать фильтры (без знания Python)

### Поисковые запросы

Файл: `parsers/hh.py`, два списка в начале файла.

**SEARCH_QUERIES** — точные запросы. hh.ru ищет по всему тексту вакансии:
```python
SEARCH_QUERIES = [
    "UX researcher",
    "CX-исследователь",
    ...
]
```

**TITLE_ONLY_QUERIES** — широкие запросы. hh.ru ищет только по заголовку (чтобы не тянуть мусор):
```python
TITLE_ONLY_QUERIES = [
    "маркетинговые исследования",
    "количественные исследования",
    ...
]
```

Добавить запрос = добавить строку в кавычках с запятой. Удалить = удалить строку.

### Белый список (что пропускаем)

Файл: `filters/stopwords.py`, список `WHITELIST`:
```python
WHITELIST = [
    "исследователь",
    "researcher",
    "research",
    "ux",
    "cx",
    ...
]
```

Если слово из этого списка есть в заголовке вакансии — она проходит.

### Чёрный список (что блокируем даже если есть слово из белого)

Там же, список `BLACKLIST`:
```python
BLACKLIST = [
    "инженер-исследователь",
    "UX/UI-дизайнер",
    ...
]
```

### Чёрный список компаний

Там же, `COMPANY_BLACKLIST` — сюда можно добавлять спамерские компании.
